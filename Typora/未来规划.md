## 未来规划

### 程序员的路径

```
1.程序→工程师→架构或更专业的工程师。
2.程序→组长→项目经理。
3.程序→工程师→技术总监→管理层。
4.粗略的几个方向。。
```

### 规划

```
1.在工作中，向前辈学习，提高自己的技术水平，积累自己的开发经验
2.业余时间的话，研究下各种技术的实现细节吧，看些专业书籍，写些博客，逐步摸清自己未来的发展方向，不出意外的话，可能技术，可能管理。可能会向着架构师或技术管理方面发展吧，不想当架构师的程序员不是一个好的程序员
3.“代码大全”中说：“深入一门语言编程，不要浮于表面”，更多是要学习设计思想吧
4.，花时间去研究一下MFC、VCL、J2EE、.Net它们框架设计或者源码；除了会用J2EE、 JBoss、Spring、Hibernate等等优秀的开源产品或者框架，抽空看看大师们是如何抽象、分析、设计和实现那些类似问题的通用解决方案的
5.刚来公司的，很幸运的就被分到现在这个部门，刚成立不久的，有挑战，有机遇，更加幸运的一点的是，试用期间，就跟随我的导师，一起去华为出差了，那半个月，工作的非常辛苦，同时也非常开心，因为知道自己在做着一份有意义的工作吧。有解决问题的喜悦，有与同事协同攻关的艰辛，也有遇到困难和挫折时惆怅，体会到了同事们对这份跟工作的热情和重视，能在岗位上体现人生价值。有他们自己坚守，我感触挺深的
6.在工作非常辛苦的时候，帮我梳理工作上遇到的问题，，分享他刚工作上的一些故事，和解决问题的方法，对我来说，这有着很好的借鉴和引导作用
```
